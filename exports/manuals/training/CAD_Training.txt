=== Slide 24 ===
Track Line Display
Status Bar
LL:
Left limit of a currently selected track segment
RL:
Right limit of a currently selected track
USER:
Name of the user logged in and having control of the territory
X:
The X coordinate of the current mouse cursor position on the track line display. The left side X = 0, and the right X = 1898
Y:
The Y coordinates the current mouse cursor position on the track line display. The top Y = 0, and the bottom Y = 1073
Date and time:
The time and date from the system clock
H:
The number of handles the application is currently using
M:
The amount of computer memory that the application is currently using
CONNECTED TO:
This shows that the workstation is connected to the database server
FEEDBACK:
Shows feedback to the dispatcher

TMDS Functions
Tool Tips
Hovering over a track segment, switch (point), or signal causes a tooltip to be displayed that provides information about that element.
Hovering over a track segment containing bulleting restrictions, device blocks, track tags, and/or track and time authorities will provide information about those, along with the track segment information.

=== Slide 38 ===
TMDS Functions
Track Tags:
Speed
– Contains information about speed restrictions that must be communicated to a. maintenance personnel and train crews who enter the speed restriction's limits.
Highway Xing (Crossing)
– Contains information about a failure or malfunction of a highway crossing safety device that must be communicated to a. maintenance personnel and b. any train crew before they reach that crossing location.
Informational
– Contains information and reminders that are useful to the dispatcher regarding conditions other than a speed restriction or highway crossing protection system issue and which may need to be conveyed to maintenance personnel or a train crew.

=== Slide 39 ===
TMDS Functions
Device Blocks:
When a device block is applied on a track, the track is colored lavender
If the dispatcher attempts to line a signal through the track block, the feedback message will display a
BLOCK IN ROUTE
message, and the route request will be canceled.
