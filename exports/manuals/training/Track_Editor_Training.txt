Territory Basics

=== Slide 14 ===
A Territory
is the base component of the TMDS track line display.
Territories are containers for Control Points, which in turn are containers for all track-line devices(signals, switches, tracks, and other miscellaneous devices).
When the dispatcher logs in and selects a territory, they are loading several smaller territories.
Territories have a maximum size of 1920x1200, which is the size of a single dispatch monitor.
Depending the number of track components, we can combine two territories to display on a single dispatch monitor.
Each territory is assigned a three-digit
Unique ID (UID).
All components placed on a territory will automatically be assigned a six-digit UID with the first three digits containing the territory UID.
For example, all components on territory number 710 will be assigned UIDs from
710000-710999.
UIDs 0-99 are reserved for Control Points and 100-999 are for all other components.

=== Slide 15 ===
Territories are comprised of:
Control Points
: an addressable collection of devices that encompasses one or more Code Stations.
Devices:
Track, signals, switches, Misc Devices, MODs, etc. All devices MUST be assigned to a Control Point.
Code Stations
: logical group set of Controls and Indications for a corresponding programmable logic controller at the wayside. (aka – Code Function Assignments).
Code Function Assignments (
CFAs) are the control and indication mnemonic bits that are assigned to a Code Station. These control and indication bits are then assigned to track line devices to enable functionally with field devices.
