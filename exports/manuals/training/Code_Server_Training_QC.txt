Code Server
The Code Server provides for control and indication functionality between TMDS and the various legacy field signal system code lines. The packet switch provides the protocol emulation for the many code line variations in use in the industry today. The Code Line Interface Service provides a graphical directory view of the code lines, control stations, and associated control/indication bits. Furthermore, it provides the standard tools (e.g., recalls, resends, inhibiting, etc.) required to manage the code line from an administrator’s perspective.

Code Server Overview
Code Server
All wayside communication messaging is channeled through an array of code servers. Each server has a warm redundant backup for failover purposes. The redundant backup is the secondary/offline code server. The code servers maintain persistent connections to an array of packet switch devices. Packet switch devices are communication equipment such as protocol converters that process data sent to/from the field devices.
Each physical territory loaded and running on the different clients will automatically detect and subscribe to the appropriate code server(s) in order to process controls and indications for the respective code stations. A code station is an entity that contains the communication settings. This includes station addresses and control and indication values. There are other system services that also subscribe to the wayside communication server through the control server in order to read real time device status and train movement.

The Function Assignments Window
This window will display the CONTROLS and INDICATIONS for the selected CODE STATION.
The window is divided into three sections.
The top section identifies the control station and code line
The left section displays all of the CONTROLS assigned to the control point
The right section displays all of the INDICATIONS assigned to that control point
A drop-down menu selection under
Options
at the top of the window allows the window to be closed by left clicking on
EXIT
.
