1. PTC ON-BOARD SEGMENT – ENERGY MANAGEMENT (EM) MESSAGES 
1.1 Scope and Introduction 
This document contains messages passed between a PTC On-board Segment and an Energy Management (EM) system that operates on a 
locomotive. The PTC on-board segment connects to Energy Management as an EMP client and Energy Management acts as an EMP server.  
The port number used for the Class D link connection is 10050. 
 
The interchange of the PTC Status Data (0500) message and EM Status (0600) message are ‘heartbeat’ messages between the PTC on-board 
segment and Energy Management.  Using these messages, the PTC on-board segment and Energy Management negotiate the message 
“interface version” and determine which functions are supported and in operation between the two systems. 
 
Other messages exchanged between PTC and EM support the exchange of track data information, restriction information, and feedback to the 
crews.

1.5.1 L2I218 PTC Status Data (0500) - Version 5 
This message conveys status of PTC data to Energy Management.  PTC and Energy Management negotiate and reach agreement on the 
“Interface version” (achieved through 0500-0600 message exchanges) before PTC sends any other types of messages to Energy Management.  
The fields “Summary train consist CRC”, “Subdivision ID”, “Subdivision version”, “Subdivision CRC”, “Restriction Aggregate CRC”, and 
“Locomotive run status ID” must match with the EM data for these fields, or else PTC and Energy Management are out-of-synchronization 
(OOS).  When OOS Energy Management needs to obtain the data needed to become synchronized with PTC for consistent operation of the 
two systems.  Energy Management is responsible for determining the correct operating behavior when it is OOS with PTC.

1.9 Definitions 
1.9.1 On-Board Segment State 
1.9.1.1 Self Test 
A state in which the on-board segment performs a self-test of certain of its hardware and software components.  The on-board segment enters 
this state on power-up or reboot.  After a successful self test, the on-board segment transitions to CUT OUT. 
1.9.1.2 Failed 
A state in which a hardware or software error has been detected.  The on-board segment transitions to FAILED when a failure has been 
detected in any other state. 
1.9.1.3 Cut Out 
A state in which the on-board segment has not yet initialized or has detected a condition that does not allow it to enforce.  The on-board 
segment transitions to the CUT OUT state: 
 
a. From SELF TEST 
 After a successful self test 
b. From INITIALIZING, DISENGAGED, ACTIVE or RESTRICTED 
 Upon a command from a BOS 
 Locomotive air brake system is cut out 
 I-ETMS penalty brake cut out switch in “Cut Out” position 
c. From DISENGAGED, ACTIVE or RESTRICTED 
 Via crew key press to cut out or log off 
d. From INITIALIZING 
 After an unsuccessful initialization 
e. From INITIALIZING, DISENGAGED or ACTIVE 
 After an unsuccessful departure test 

PTC On-board – Energy Management ICD, Rel. 1.9, 4/7/2017 
-77- 
f. From FAILED 
 When all failures are cleared in FAILED state 
1.9.1.4 Initializing 
A transitional state initiated by the crew in which the on-board segment validates its configuration and the crew credentials and obtains essential 
information about the train.  The on-board segment transitions to this state from the CUT OUT, DISENGAGED and ACTIVE states via crew key 
press. 
1.9.1.5 Disengaged 
A state in which the on-board segment has initialized but is not prepared to enforce.  The on-board segment transitions to this state: 
 
a. From INITIALIZING 
 After successful initialization has successfully completed 
b. From ACTIVE 
 When the on-board segment is not prepared to enforce 
c. From RESTRICTED 
 Upon crew key press 
1.9.1.6 Active 
A state in which the on-board segment is prepared to enforce.  The on-board segment transitions to this state: 
 
a. From DISENGAGED 
 When the on-board segment is prepared to enforce 
1.9.1.7 Restricted 
A state in which the on-board segment enforces restricted speed but is not otherwise prepared to enforce.  The on-board segment transitions to 
this state from the ACTIVE state via crew key press.

2. Appendix A: Enumerated Message Rules: 
The following rules will be applied to the use of Enumerated Messages: 
1. Each line within the Enumerated Message will be a maximum of 16 characters followed by a linefeed character to separate lines to be 
displayed. 
2. Each Enumerated Message shall end with a NULL character. 
3. The Enumerated Message will allow only predefined insertion types to be included in the displayed message.  All submitted 
Enumerated Messages must comply with the defined format for insertable characters. 
a. When the Enumerated Message contains a ‘%T’ the displayed Enumerated Message will insert ‘NX’ where ‘X’ is the throttle 
notch value provided in the request message.  The ‘%T’ counts as 2 printable characters in the character count. 
b. When the Enumerated Message contains a ‘%B’ the displayed Enumerated Message will insert ‘DX’ where ‘X’ is the dynamic 
braking notch value provided in the request message.  The ‘%B’ counts as 2 printable characters in the character count. 
c. When the Enumerated Message contains a ‘%D’ the displayed Enumerated Message will insert ‘XY’ where ‘XY’ is the base 10 
value of the countdown timer value provided in the request message.  The ‘%D’ counts as 2 printable characters in the 
character count. 
d. When the Enumerated Message contains a ‘%P’ the displayed Enumerated Message will insert ‘XY’ where ‘XY’ is the base 10 
value of the brake pressure value provided in the request message.  The ‘%P’ counts as 2 printable characters in the character 
count. 
Note that there are no units or braking type associated with this brake pressure insertion.  This must be included in the 
Enumerated Message. 
4. Invalid Enumerated Message requests will generate an acknowledgement message stating an invalid display request message was 
sent. 
5. An Enumerated Message request containing a countdown timer will have the countdown timer displayed as the message time to live 
value. 

PTC On-board – Energy Management ICD, Rel. 1.9, 4/7/2017 
-79- 
6. When the countdown timer within an Enumerated Message reaches zero the message will be displayed for one second with a 
countdown value of zero before being removed. 
7. Enumerated Messages can be cancelled by sending an Enumerated Message display request with the overlay message type 
enumeration value set to zero or overwriting the ITF/BCP Overlay with another displayed message. 
8. Enumerated Messages shall not be brand or product specific. 
9. Use the term ‘AUTO’ to refer to the EM state where EM is controlling the locomotive’s throttle. 
10. Use the term ‘MANUAL’ to refer to the EM state where EM is not controlling the locomotive’s throttle.
